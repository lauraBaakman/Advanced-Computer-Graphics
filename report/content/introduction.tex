% \todo[inline]{Context: Why PN triangles: fast smooth real-time rendering}
\textcite{vlachos2001curved} introduced curved point-normal triangles as a method to improve the visual appeal of triangular meshes in real-time entertainment. A major advantage of this method at the time of introduction (\citeyear{vlachos2001curved}) is that it is a fast method to improve the visual quality of the rendered objects.

% \todo[inline]{Context: global idea of PN triangles}
If point-normal (PN) triangles are used, each triangle of the mesh is replaced by a curved shape, that is tessellated into a user selected number of smaller flat triangles. Each point-normal triangle consists of a geometric and a normal component. These components are respectively defined as a cubic and a quadratic BÃ©zier patch. The coefficients of the patches are based on the vertices and the normals of the corresponding triangle in the input mesh.

% \todo[inline]{Application Domain: useful for real time rendering, not for CAD, explain why not....}
Although curved point-normal triangles improve the visual appeal of a rendered mesh via ``smoother silhouettes and better shading" \cite{vlachos2001curved} and are fast enough for real-time rendering, their application domain is restricted to entertainment. They should not be used in, for example computer-aided design. Since the rendering gives the model a visual smoothness that is not present in the actual mesh.
